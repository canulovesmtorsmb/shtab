# ШТАБ.EXE – итоговый план и отчёт по доработке

Дата: 22 июня 2026. Формат соблюдает правила проекта: короткое тире, без интерпункта, без эмодзи.

## 1. Что это и с чего начали

ШТАБ.EXE – личный командный центр (ученики, занятия, заказы, журнал ИТМО, AIM, блогинг, орг, арсенал). Одностраничное офлайн-приложение: данные в браузере (IndexedDB), синхронизация между устройствами через свой Cloudflare Worker + база D1. По итогам аудита выбраны: редизайн в неоморфизм, фикс багов, лента комментариев, безопасная синхронизация, голосовой ассистент. Данные критичны (5 из 5).

## 2. Файлы в архиве

- shtab.html – боевое приложение со всеми правками. На GitHub залить как index.html.
- worker.js – обновлённый Cloudflare Worker (безопасное слияние + эндпоинт ассистента /ask). В shtab-sync/src/worker.js, затем deploy.
- shtab-neo-prototype.html – кликабельный прототип неоморфизма (эталон стиля).
- ТЗ-Штаб-аудит.md – техзадание по итогам аудита.
- ПЛАН-Штаб-итог.md – этот документ.

## 3. Что сделано (по спринтам)

Спринт 1 – навигация и расписание: фикс "выкидывает на главную" (функции editFrom и goBack, возврат на исходный экран после сохранения/отмены/удаления, 12 точек входа); почасовое расписание.

Спринт 2 – drag-and-drop и синхронизация: надёжный перенос (порог 6px, захват указателя, автоскролл); перенос одной даты повторяющегося события без потери данных (исключение + одноразовая копия, серия цела); автосинхронизация ощущается живой (pull при возврате в приложение, индикатор СИНХР в строке состояния).

Спринт 3 – неоморфизм: две темы neo (светлая) и neo-dark (тёмная) в переключателе. Классический и старый тёмный скины убраны из обращения, по умолчанию неоморфизм светлый.

Лента комментариев: новый тип поля – датированная лента у ученика, занятия и события. Добавление и удаление, история не перезаписывается. Прежние поля заметок сохранены.

Безопасное слияние синхронизации: Worker сливает состояние по записям (id + updatedAt), правки с двух устройств не затирают друг друга. Удаления через надгробия (deleted) без воскрешения. Оптимизация: флаг changed – клиент переписывает локальную базу только при реальных чужих изменениях.

Фиксы по ревью: открытый редактор не стирается фоновой синхронизацией (editorOpen); drag по часам.

Почасовые виды: недельный вид – полноценная почасовая сетка (часы по вертикали, 7 дней по горизонтали, перенос между днями и часами), день – тоже почасовой. Текст часовой сетки тёмный (var(--ink)).

## 4. Как выкатить (порядок важен)

1. Бэкап: открыть текущий сайт, нажать Экспорт, сохранить файл с данными.
2. Worker: положить новый worker.js в shtab-sync/src/worker.js, в папке shtab-sync выполнить npx wrangler deploy. Схему базы менять не нужно.
3. Проверить на запасном ключе-пространстве: создать новое пространство, поправить и поудалять записи с двух вкладок, убедиться, что ничего не теряется.
4. Сайт: переименовать shtab.html в index.html, залить поверх в репозиторий, Commit. Подождать минуту, обновить с жёстким сбросом кеша.
5. iPhone: удалить старый ярлык с экрана Домой и добавить заново.

Совместимость: новый сайт работает и со старым Worker, новый Worker не ломает старый клиент.

## 5. Что проверено

- Точечные правки кода с контролем количества замен, ноль лишних совпадений.
- Синтаксис всех скриптов (shtab.html, worker.js) – чисто.
- Логика слияния синхронизации – 14 юнит-тестов в Node плюс прогон на реальном бэкапе.
- Сквозной прогон в реальном браузере с IndexedDB и D1 в песочнице недоступен (нет сети к npm), финальная проверка – на твоей стороне.

## 6. Известные ограничения

- Неоморфизм наложен поверх старой оконной разметки – возможны мелкие неровности, подгоняется по скриншотам.
- Надгробия удалений хранятся в localStorage. Полная очистка данных браузера сбрасывает их (бэкап данных это не затрагивает).
- Комментарий к повторяющемуся событию общий для всей серии (у занятий поле на каждом занятии).

## 6a. Голосовой ассистент (вариант Б, с нейросетью)

Что добавлено: в правом нижнем углу круглая кнопка ИИ. По клику открывается панель – кнопка Голос (распознавание речи), поле ввода и чат. Ответ зачитывается вслух. Ассистент отвечает по твоим реальным данным: занятия на сегодня, ближайшие занятия и события, ученики, долги, заказы, открытые задачи. Мозг работает через твой Worker: сайт зовёт эндпоинт /ask, Worker обращается к нейросети, ключ хранится секретом на сервере и в браузере не светится. CSP сайта менять не нужно.

Как включить мозг (один раз):
1. Получи бесплатный ключ. Проще всего Groq: console.groq.com, раздел API Keys, создать ключ.
2. В папке shtab-sync задай секрет и передеплой: npx wrangler secret put AI_API_KEY (вставь ключ, Enter), затем npx wrangler deploy.
3. По умолчанию Worker ходит в Groq, модель llama-3.3-70b-versatile. Другой провайдер (OpenRouter и пр.) – задай секреты AI_API_BASE и AI_MODEL (OpenAI-совместимый формат), затем deploy.

Честные ограничения:
- Пока ключ не задан, ассистент честно скажет, что мозг не подключён (приложение при этом работает).
- Данные вопроса (имена, цифры) уходят к провайдеру нейросети. Для личного инструмента это нормально, но знать стоит.
- iPhone/Safari: голосовой ввод часто не работает (озвучка и ввод текстом работают). Надёжно слушает Chrome на компьютере и Android.

## 6b. Ассистент — этапы 1–7 (что умеет)

Этап 1. Память диалога, подтверждения действий кнопками в чате, режим «только чтение», журнал действий, кнопка «стоп речь».
Этап 2. Действия с подтверждением и откатом: добавить/перенести/изменить занятие, отметить проведённым/оплаченным, добавить ученика, абонемент, заказ, событие (в т.ч. повтор), задачу; закрыть задачу; отметить привычку; отмена последнего действия; проверка накладок по времени.
Этап 3. Точная аналитика в контексте: доход за день/неделю/месяц (получено и долг), запланировано, кто не занимался более 3 недель, накладки. Цифры считаются в приложении, не нейросетью.
Этап 4. Уточняющие вопросы при неоднозначности; генерация текста сразу в поле (ДЗ, прогресс, заметки) через set_field; итоги недели из аналитики.
Этап 5. Утренняя сводка дня (баннер раз в день при открытии): занятия, дедлайны до 3 дней, абонементы на исходе; озвучка по кнопке.
Этап 6. Интеграции: подготовить письмо/отчёт с кнопкой Копировать (copy_text); добавить событие в календарь устройства (make_ics); статистика соцсети по ссылке (social).
Этап 7. Настройки в панели ассистента: выбор модели ИИ (без правки Worker, ключ остаётся секретом на сервере) и режим «компактный контекст» для экономии токенов.

Безопасность: любое изменение данных требует подтверждения кнопкой; удалять голосом нельзя; есть откат последнего действия и режим только чтение.

Про локальную модель (Ollama): встроить в статический сайт нельзя — браузер из HTTPS-страницы не достучится до localhost (CSP и mixed content), а облачный Worker тем более не видит твой компьютер. Это возможно только если запускать ШТАБ локально и настроить CORS у Ollama — отдельная история, не для GitHub Pages.
